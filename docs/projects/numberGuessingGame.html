Number Guessing Game Project

Summary

We all have heard of the game that goes something like this, "Guess a number between 1 and 100." Well, in this project, you will re-create this game in JavaScript.

Project Requirements

The application is required to have a home.html page with a js and css file for it. To ensure that it is ready to add more functionality, you should design the application file structure in such a way that you are able to add more pages with ease. The application:

  • Has a professional look and feel

  • Includes an html form with the following fields:

    • Low number for range (guess a number between 1 and 100, this input will allow the user to pick the bottom number of this range for the game)

    • High number for range

    • The form will look something like this: 

      
<form name="guessForm" onsubmit="submitData(); return false">
    <input type="text" name="min">
    <input type="text" name="max">
    <button type="submit">Play Game!</button>
</form>

  • A button to submit the form

  • Implements the following functions:

    • submitData(). This function will retrieve the form data, and check for validation errors (they need to submit valid numbers AND the low number has to be less than the high number). If there is an error, display the error by calling the updateDOM function and stop the remaining code execution. If there is no error, call the numberGuessGame and pass in min and max as parameters.

    • updateDOM(value, color). This function will display the error message if there is a validation error. And later it will also display "CONGRATULATIONS!!! __ is correct!". The value parameter will either be the error message string, or the congratulations message. The color you can use to make the error red, or the congrats message a special color.

    • numberGuessGame(min, max). This function will include the loop and the logic to drive the game. Use a do while loop with a condition similar to while(userGuess != actualNumber). Inside the loop, use prompt() to prompt the user for their next guess, and display their progress letting them know if they need to guess higher or lower in the prompts.
